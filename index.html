Why Schemock

Built for the frontend dev blocked by backend

Stop waiting. Stop hardcoding. Stop using stale mock data that drifts from reality.

📋
Schema is truth

JSON Schema is your contract. Use it to generate the mock, validate requests, and document the API — one source of truth.

⚡
Zero friction

Download the binary. Run schemock start. That's it. No Node, no Docker, no sign-up, no config files.

🔄
Hot reload

Edit your schema, save — the server auto-restarts. Test schema changes instantly without restarting your dev server.

🎭
Realistic data

UUIDs, emails, dates, enums — generated to match your schema formats. Not "John Doe 1, John Doe 2."

🧪
Test scenarios

Flip between slow networks, error-heavy responses, and happy-path with a single flag. Test your UI's edge cases.

🤖
AI-native (MCP)

Built-in MCP server. Let Claude, Cursor, or any MCP AI explore your mock API and generate matching code.

🤖 MCP Native

Your AI assistant
uses your mock API

Schemock ships with a built-in MCP server. Your AI tools can discover endpoints, make requests, and generate matching frontend code — all from your JSON Schema.

  • Claude, Cursor, and any MCP client work out of the box
  • Auto-discovers routes from your schema
  • AI generates type-safe React/Vue components from responses
  • No configuration needed
// Claude Desktop / Cursor — add to your MCP config {
  "mcpServers": {
    "schemocker": {
      "command": "node",
      "args": [
        "/path/to/schemocker/dist/mcp-server/index.js"
      ]
    }
  }
}

// Then ask your AI:
"Generate a React component
  fetching from /api/data"
Testing Scenarios

Test how your UI handles reality

Flip between network conditions and error states without touching your code.

🌿 Happy Path
Fast responses, no errors. Default mode.
--scenario happy-path
🐢 Slow Network
1–3 second delays. Test your loading states.
--scenario slow
💥 Error Heavy
Random 400s and 500s. Test your error boundaries.
--scenario error-heavy
😰 Sad Path
Slow + errors combined. Worst case.
--scenario sad-path
